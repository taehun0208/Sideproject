# mio stamp — 프로젝트 문서

---

## 왜 만들었나

미용실 단골 손님에게 "5번 오시면 1번 무료"를 약속하는 건 흔한 일이다.  
그런데 대부분의 미용사는 그 약속을 종이 스탬프 카드에 도장 찍는 방식으로 지킨다.

문제는 손님이 카드를 잃어버리거나, 미용사가 스탬프 도장을 챙겨야 하거나,  
결국 흐지부지되어 "그냥 됐어요"로 끝나는 경우가 너무 많다는 것.

**mio stamp**는 그 불편함을 없애기 위해 만든 앱이다.  
종이도, 도장도, 앱 설치도 필요 없다.  
미용사(mio쌤)가 시술할 때마다 버튼 하나를 누르면 끝이고,  
손님은 링크 하나로 자신의 스탬프 현황을 언제든 확인할 수 있다.

---

## 누가 쓰는가

| 역할 | 설명 |
|------|------|
| **미용사 (mio쌤)** | 앱의 주 사용자. 로그인 후 고객을 관리하고 시술을 기록한다 |
| **손님** | 앱 설치 불필요. 미용사가 공유한 링크로 자신의 스탬프 카드를 확인한다 |

---

## 핵심 흐름

```
[미용사 로그인]
     ↓
[고객 검색 또는 등록]
     ↓
[시술 종류 선택 → 탭 한 번]
     ↓
[자동으로 시술 기록 + 5회 달성 시 쿠폰 자동 발급]
     ↓
[공유 링크 생성 → 손님에게 카카오톡 등으로 전달]
     ↓
[손님은 링크 열면 자신의 스탬프 카드 확인 가능]
```

손님 입장에서는 앱 설치, 회원가입, 로그인이 **전혀 필요 없다**.  
링크만 열면 된다.

---

## 주요 기능

### 미용사 화면 (로그인 필요)

**대시보드**
- 전체 고객 수 / 총 시술 횟수 / 활성 쿠폰 수를 한눈에 확인
- 단골 손님 목록, 신규 손님 등록으로 바로 이동

**고객 관리**
- 이름 또는 전화번호로 실시간 검색
- 고객 등록 시 이름과 전화번호만 입력 (최소한의 정보)

**고객 상세 (핵심 화면)**
- 시술 종류별(다운펌 / 염색 / 클리닉 / 커트) 스탬프 현황 표시
- 버튼 탭 한 번으로 시술 기록
- 5회 달성 시 해당 시술 쿠폰 자동 발급 + 알림
- 시술 히스토리 전체 조회
- 손님 전용 공유 링크 생성 및 복사

### 손님 화면 (로그인 불필요)

- 토큰 기반 URL(`/c/[token]`)로 접근
- 시술 종류별 스탬프 카드 (5칸 원형 스탬프)
- 사용 가능한 쿠폰 / 사용 완료 쿠폰 목록
- "미용사에게 이 화면을 보여주세요" 안내

---

## 쿠폰 발급 로직

```
시술 기록 → 해당 시술 누적 횟수 확인
    ↓
횟수 % 5 === 0 이면 → 이미 발급된 활성 쿠폰이 없을 때만 신규 발급
    ↓
쿠폰 status: 'active' 로 저장 → 손님 화면에 표시
```

중복 발급 방지 로직이 포함되어 있어, 같은 시술로 활성 쿠폰이 이미 있으면 추가로 발급되지 않는다.

## 기술 스택

| 항목 | 사용 기술 |
|------|----------|
| 프레임워크 | Next.js (App Router) |
| 언어 | TypeScript |
| 스타일링 | Tailwind CSS |

| 백엔드 / DB | Supabase (PostgreSQL) |
| 인증 | Supabase Auth (Email / Password) |
| 보안 | Row Level Security (RLS) |
| 폰트 | Bebas Neue, Dancing Script, Geist |

---

## 데이터 구조

```
auth.users (Supabase 관리)
    └── customers (미용사의 고객 목록)
            ├── treatments (시술 기록)
            ├── coupons (발급된 쿠폰)
            └── tokens (손님 공개 링크용 토큰)
```

**보안 설계 (RLS)**
- 미용사는 자신이 등록한 고객 데이터만 읽기/쓰기/삭제 가능
- 손님(비로그인)은 토큰에 연결된 단 한 명의 고객 데이터만 읽기 가능
- 쿠폰 생성·수정은 미용사만 가능

## 시술 종류

현재 지원하는 4가지 시술 카테고리:

| 시술 | 아이콘 |
|------|--------|
| 다운펌 | 〰️ |
| 염색 | 🎨 |
| 클리닉 | ✨ |
| 커트 | ✂️ |

각 시술은 **독립적으로 스탬프가 적립**된다.  
예: 염색 5회 달성 → 염색 쿠폰 1장 / 커트는 별도 카운트

## 디자인 방향

사용자(mio쌤)는 20대 미용사다.  
투박한 어드민 느낌이 아닌, **감각적이고 트렌디한 느낌**을 의도했다.

- **네이비(#1A2855) + 크림(#FAF8F4) + 로즈(#E8A0AD)** 컬러 팔레트
- **Bebas Neue** — 숫자와 제목에 임팩트 있는 디스플레이 폰트
- **Dancing Script** — "mio" 브랜드 네임에 필기체 감성
- **바텀시트 로그인** — 모바일에서 앱스러운 UX, 태블릿에서는 좌우 분할
- **네이비 헤더 + 크림 바디** — 페이지마다 일관된 레이어드 레이아웃
- 반응형: 모바일 기준 설계, 태블릿/패드에서도 자연스럽게 대응

## 환경 설정

`.env.local` 에 아래 값이 필요하다:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...
```

Supabase 대시보드 → **Settings → API** 에서 확인.

## 실행

```bash
npm install
npm run dev
# → http://localhost:3000
```

로그인 후 `/admin` 에서 시작.
