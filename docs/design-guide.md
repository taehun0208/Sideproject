# Design & Development Guide

## Week 1 개발 팁

### 테마 및 색상 설정
- **테마:** Dark 모드 기본 설정
- **포인트 컬러:** 형광 연두 (#00FF66) 또는 일렉트릭 블루로 `globals.css`에 지정
- 이것만으로도 2030 타겟의 힙한 테크 감성이 즉시 드러남

### 초기 개발 순서
1. shadcn/ui Dark 테마 적용
2. 기본 컴포넌트만 배치 (Button, Card, Input)
3. `admin/customers/[id]` 의료 차트 스타일 그리드 레이아웃
4. 시술 버튼 4개 배치 + 클릭 시 카운트 증가 애니메이션
5. 손님 공개 페이지 프로그레스 바 UI

## 디자인 참고 자료

### shadcn/ui Blocks
- URL: ui.shadcn.com/blocks
- 용도: 어드민 페이지 전체 레이아웃 (사이드바, 검색창, 통계 카드)
- 특징: 미니멀 테크 디자인의 정석, 그대로 복사해서 사용 가능

### v0.dev
- URL: v0.dev
- 용도: AI 기반 Tailwind UI 생성
- 예시 프롬프트:
  - `"Medical style customer dashboard, cyberpunk neon accent, dark mode"`
  - `"Sleek coupon stamp board for mobile web, digital progress bar"`
- 특징: shadcn + Tailwind 코드로 즉시 사용 가능한 UI 생성

### Mobbin
- URL: mobbin.com
- 용도: 실제 앱 UI 레퍼런스
- 참고 대상: 토스, 애플카드, 피트니스 앱의 대시보드 및 멤버십 카드 화면
- 특징: 2030 남성들이 깔끔하다고 느끼는 그리드 레이아웃과 폰트 밸런스

### Godly Website
- URL: godly.website
- 용도: 강렬한 비주얼 & 트렌디한 웹 사이트 큐레이션
- 특징: 과감한 폰트, 블랙&화이트에 네온 컬러 조합, 스트리트 패션 브랜드 스타일
- 활용: 손님 공개 페이지(`/c/[token]`) 비주얼 톤앤매너 참고

## 확인사항

### ⚠️ 필독

**한 손님의 스탬프 구조가 어떻게 되나요?**

- **다중 스탬프 구조**: 다운펌 스탬프 따로, 염색 스탬프 따로 관리
  - DB 설계: `treatments.treatment_type` 별로 Count 계산
  - 장점: 고객이 선호 시술별로 쿠폰 수집
  - 복잡도: ⭐⭐⭐

- **단일 스탬프 구조**: 시술 종류 상관없이 방문할 때마다 카운트 증가
  - DB 설계: 모든 시술을 하나의 카운터로 통합
  - 장점: 미용사 입력 로직 단순, 손님도 이해 쉬움
  - 복잡도: ⭐

위 내용에서는 **다중 스탬프 구조**로 가정해 설계했습니다. 어떤 방식으로 진행하실 예정인지 확인해주세요!
